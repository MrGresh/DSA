#### **Goal:** Master the "First Instinct." This guide maps common problem descriptions to their most efficient algorithmic patterns.
#### Use this as a high-level mental map during the first 2 minutes of any technical interview.

---

## 🛠️ The Pattern Catalog

### 1. Pattern: Two Pointers
* **Description:** Uses two pointers to traverse a data structure from different ends or directions.
* **Usage:** Best for ordered data structures (sorted arrays/lists) where decisions depend on pointer positions.
* **Problems:** 'Pair with Target Sum', 'Remove Duplicates', 'Squaring a Sorted Array'.

### 2. Pattern: Island (Matrix Traversal)
* **Description:** Traversing a matrix to find contiguous groups of elements (islands).
* **Usage:** Grid-based problems requiring grouping of connected elements.
* **Problems:** 'Number of Islands', 'Max Area of Island', 'Flood Fill'.

### 3. Pattern: Fast & Slow Pointers
* **Description:** Two pointers moving at different speeds (e.g., $1x$ and $2x$).
* **Usage:** Cycle detection, finding middle elements, or Palindrome checks in Linked Lists.
* **Problems:** 'LinkedList Cycle', 'Middle of the LinkedList', 'Palindrome LinkedList'.

### 4. Pattern: Sliding Window
* **Description:** Creating a 'window' of size $K$ (fixed) or $X$ (variable) and sliding it across the data.
* **Usage:** Finding a contiguous subarray or substring fulfilling specific conditions.
* **Problems:** 'Maximum Sum Subarray of Size K', 'Smallest Subarray with a given sum'.

### 5. Pattern: Merge Intervals
* **Description:** Handling overlapping intervals or ranges.
* **Usage:** Problems involving time intervals, scheduling, or range sequences.
* **Problems:** 'Merge Intervals', 'Insert Interval', 'Intervals Intersection'.

### 6. Pattern: Cyclic Sort
* **Description:** Sorting an array where numbers are in a finite, known range (e.g., $1$ to $n$).
* **Usage:** Identifying missing, duplicate, or misplaced numbers in $O(n)$ time.
* **Problems:** 'Cyclic Sort', 'Find the Missing Number', 'Find all Duplicates'.

### 7. Pattern: In-place Reversal of a Linked List
* **Description:** Reversing nodes without using extra memory.
* **Usage:** Reversing sequences or sub-sections of a list in $O(1)$ space.
* **Problems:** 'Reverse a LinkedList', 'Reverse a Sub-list'.

### 8. Pattern: Tree Breadth First Search (BFS)
* **Description:** Level-by-level traversal using a Queue.
* **Usage:** Shortest path in unweighted graphs or level-order tree processing.
* **Problems:** 'Level Order Traversal', 'Zigzag Traversal'.

### 9. Pattern: Tree Depth First Search (DFS)
* **Description:** Exploring a branch completely before visiting siblings (Recursion/Stack).
* **Usage:** Searching deep into paths, calculating sums, or leaf-to-root properties.
* **Problems:** 'Binary Tree Path Sum', 'All Paths for a Sum'.

### 10. Pattern: Two Heaps
* **Description:** Using a Max-Heap and Min-Heap simultaneously to divide data.
* **Usage:** Finding medians in a data stream or maintaining balanced halves.
* **Problems:** 'Median of a Number Stream', 'Sliding Window Median'.

### 11. Pattern: Subsets
* **Description:** Generating all possible combinations or permutations of a set.
* **Usage:** Exploring the entire power set of a given input.
* **Problems:** 'Subsets', 'Subsets With Duplicates', 'Permutations'.

### 12. Pattern: Modified Binary Search
* **Description:** Adapting $O(\log n)$ search for non-standard sorted structures.
* **Usage:** Bitonic arrays, rotated arrays, or finding the "Ceiling" of a number.
* **Problems:** 'Order-agnostic Binary Search', 'Next Letter'.

### 13. Pattern: Top 'K' Elements
* **Description:** Using a Heap to track the most/least frequent or largest/smallest elements.
* **Usage:** Sorting-related problems where you don't need a full sort.
* **Problems:** 'Top K Frequent Numbers', 'Kth Largest Number in a Stream'.

### 14. Pattern: Bitwise XOR
* **Description:** Manipulating bits directly to find unique elements.
* **Usage:** Finding a single missing or duplicate element in an array.
* **Problems:** 'Single Number', 'Complement of Base 10 Number'.

### 15. Pattern: Backtracking
* **Description:** DFS with an "Undo" step to explore all valid combinatorial paths.
* **Usage:** Puzzles, games, or generating valid parentheses.
* **Problems:** 'Sudoku Solver', 'N-Queens', 'Generate Parentheses'.

### 16. Pattern: 0/1 Knapsack (Dynamic Programming)
* **Description:** Optimization where items have weights and values; decisions are "take" or "leave".
* **Usage:** Resource allocation or subset sum problems under constraints.
* **Problems:** '0/1 Knapsack', 'Equal Subset Sum Partition'.

### 17. Pattern: Topological Sort (Graph)
* **Description:** Linear ordering of nodes where $u \rightarrow v$ implies $u$ comes before $v$.
* **Usage:** Dependency scheduling, build systems, or prerequisite ordering.
* **Problems:** 'Task Scheduling Order', 'Alien Dictionary'.

### 18. Pattern: K-way Merge
* **Description:** Merging $K$ sorted inputs into one using a Heap.
* **Usage:** Efficiently combining multiple sorted lists or arrays.
* **Problems:** 'Merge K Sorted Lists', 'Smallest Number Range'.

### 19. Pattern: Monotonic Stack
* **Description:** Stack that maintains strictly increasing or decreasing order.
* **Usage:** Finding the "Next Greater" or "Next Smaller" element in $O(n)$.
* **Problems:** 'Next Greater Element', 'Largest Rectangle in Histogram'.

### 20. Pattern: Multi-threaded
* **Description:** Algorithms designed for parallel execution.
* **Usage:** Dividing tasks into independent sub-tasks for concurrent processing.
* **Problems:** 'Invert Binary Tree', 'Binary Search Tree Iterator'.

### 21. Pattern: Union Find (DSU)
* **Description:** Data structure tracking partitions of a set into disjoint groups.
* **Usage:** Connectivity problems, checking for same-group membership, or cycle detection in undirected graphs.
* **Problems:** 'Redundant Connection', 'Number of Provinces'.
