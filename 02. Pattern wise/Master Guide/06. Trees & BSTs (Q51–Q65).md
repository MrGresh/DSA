# 🌳 Section 6: Trees & BSTs (Q51–Q65)

#### **Core Concept:** Trees are almost always solved recursively. The secret to success is clarity: define exactly what your function returns and trust the recursion to handle the subtrees. 
#### **BST Rule of Thumb:** The **Inorder traversal of a BST is always sorted**—this simple fact unlocks the majority of BST-specific problems.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 51 | **Invert Binary Tree** | DFS / BFS | Swap left and right child at every node | $O(n) / O(h)$ |
| 52 | **Max Depth of Binary Tree** | DFS Postorder | `1 + max(depth(left), depth(right))` | $O(n) / O(h)$ |
| 53 | **Diameter of Binary Tree** | DFS + Global Max | Return height; update `ans = left_height + right_height` | $O(n) / O(h)$ |
| 54 | **Balanced Binary Tree** | DFS w/ Height | Return height or `-1` if unbalanced; propagate up | $O(n) / O(h)$ |
| 55 | **Same Tree** | Simultaneous DFS | Compare values, recurse left and right simultaneously | $O(n) / O(h)$ |
| 56 | **Subtree of Another Tree** | DFS + isSameTree | For each node in `s`, run `isSameTree` check | $O(m \cdot n) / O(h)$ |
| 57 | **LCA of BST** | BST Property | Both < root: go left; both > root: go right; else root | $O(h) / O(1)$ |
| 58 | **LCA of Binary Tree** | DFS Postorder | LCA = first node where both subtrees return non-null | $O(n) / O(h)$ |
| 59 | **Level Order Traversal** | BFS with Queue | Track level size; process node by level | $O(n) / O(n)$ |
| 60 | **Right Side View** | BFS / DFS | Last node at each BFS level (or DFS right-first) | $O(n) / O(n)$ |
| 61 | **Good Nodes in Tree** | DFS w/ Running Max | Node is "good" if `val >= max` on root $\rightarrow$ node path | $O(n) / O(h)$ |
| 62 | **Validate BST** | DFS with Bounds | Pass `(min, max)` bounds; `node.val` must be in range | $O(n) / O(h)$ |
| 63 | **Kth Smallest in BST** | Inorder Traversal | Inorder = sorted; stop at the kth element | $O(h+k) / O(h)$ |

### 💡 Implementation Secrets
* **The "h" Complexity:** Note that space complexity is often $O(h)$ (height), not $O(n)$. In a balanced tree, $h = \log n$, but in a skewed tree, $h = n$.
* **Lowest Common Ancestor (LCA):** In a BST, the LCA is simply the "split point" where one target value is smaller than the current node and the other is larger.
* **DFS vs. BFS:** Use DFS (recursion) when you need to reach leaves or check properties of subtrees. Use BFS (Queue) for anything involving "levels" or "shortest distance."
