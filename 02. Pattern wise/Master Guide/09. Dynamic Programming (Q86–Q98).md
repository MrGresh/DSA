# 📈 Section 9: Dynamic Programming (Q86–Q98)

> #### **Core Concept:** DP = Recursion + Memoization = Tabulation.
> #### The most critical step is to **define the state first**: What does `dp[i]` represent?
> #### Write the recurrence relation on paper before touching a single line of code.
> #### For 2D DP, always ask: "What does `dp[i][j]` represent in the context of the subproblem?"

---

### 🗺️ DP Pattern Map
* **1D DP:** Fibonacci variants, House Robber, Climbing Stairs.
* **Knapsack:** 0/1 (Subset Sum) vs. Unbounded (Coin Change).
* **String DP:** LCS, Edit Distance, Palindrome problems.
* **Grid DP:** Unique Paths, Minimum Path Sum, Dungeon Game.
* **Interval DP:** Matrix Chain Multiplication, Burst Balloons.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 86 | **Climbing Stairs** | 1D DP (Fibonacci) | `dp[i] = dp[i-1] + dp[i-2]` | $O(n) / O(1)$ |
| 87 | **House Robber** | 1D DP | `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` | $O(n) / O(1)$ |
| 88 | **House Robber II** | Circular Array | Run Robber twice: once excluding 1st, once excluding last | $O(n) / O(1)$ |
| 89 | **Longest Palindromic Substring** | Expand Center | Try each center (odd/even); $O(1)$ space logic | $O(n^2) / O(1)$ |
| 90 | **Count Palindromic Substrings** | Expand Center | $2n-1$ centers; count each valid expansion | $O(n^2) / O(1)$ |
| 91 | **Decode Ways** | 1D DP | `dp[i] =` valid single + valid double digit combos | $O(n) / O(n)$ |
| 92 | **Coin Change** | Unbounded Knapsack | `dp[i] = min(dp[i], dp[i - coin] + 1)` for each coin | $O(n \cdot \text{amt}) / O(\text{amt})$ |
| 93 | **Word Break** | 1D DP + Set | `dp[i] = true` if `dp[j]` && `s[j:i]` is in wordDict | $O(n^2 \cdot m) / O(n)$ |
| 94 | **Longest Increasing Subsequence** | DP / Binary Search | DP or Patience Sort for $O(n \log n)$ optimization | $O(n \log n) / O(n)$ |
| 95 | **Unique Paths** | 2D DP / Math | `dp[i][j] = dp[i-1][j] + dp[i][j-1]` | $O(m \cdot n) / O(m \cdot n)$ |
| 96 | **Longest Common Subsequence** | 2D DP | Match: $+1$; Else: `max` of two skips | $O(m \cdot n) / O(m \cdot n)$ |
| 97 | **Edit Distance** | 2D DP | `1 + min(insert, delete, replace)` | $O(m \cdot n) / O(m \cdot n)$ |
| 98 | **Partition Equal Subset Sum** | 0/1 Knapsack | `dp[i] =` can we achieve sum $i$ from elements so far? | $O(n \cdot \text{sum}) / O(\text{sum})$ |

### 💡 Implementation Secrets
* **Space Optimization:** In almost all 1D and 2D DP problems, you can reduce space complexity (e.g., $O(n^2)$ to $O(n)$) by only keeping the previous row or previous two variables.
* **The "Base Case":** Most DP bugs come from incorrect base cases (e.g., `dp[0]`). Double-check these for $i=0$ and $i=1$.
* **Top-Down vs. Bottom-Up:** If you're struggling with the iterative (tabulation) logic, write the recursive version with a `memo` map first.
