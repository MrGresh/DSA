# 🔗 Section 4: Linked Lists (Q34–Q43)

#### **Core Concept:** Linked list questions test your ability to manipulate pointers without losing references. 
#### Master two key techniques: **Fast/Slow Pointers** (for cycles, middles, and kth-from-end) and the **Dummy Node** (to prevent edge case headaches with the head node). 
#### **Pro Tip:** Always draw the pointer states on paper/whiteboard before you start writing code.

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 34 | **Reverse Linked List** | 3-pointer iterative | `prev=null`, `cur=head`; swap then advance | $O(n) / O(1)$ |
| 35 | **LL Cycle Detection** | Fast/Slow Pointers | Meet → cycle; reset one to head → find entry | $O(n) / O(1)$ |
| 36 | **Merge Two Sorted Lists** | Dummy Node + 2 Ptr | Dummy head; compare and link smaller node | $O(m+n) / O(1)$ |
| 37 | **Remove Nth Node from End** | Fast/Slow w/ N-gap | Move fast N steps, then both move together | $O(n) / O(1)$ |
| 38 | **Reorder List** | Mid + Reverse + Merge | Find mid, reverse 2nd half, merge alternately | $O(n) / O(1)$ |
| 39 | **LRU Cache** | HashMap + DLL | Map `key -> node`; Doubly LL for $O(1)$ move-to-front | $O(1)$ ops / $O(n)$ |
| 40 | **Merge K Sorted Lists** | Min-Heap size K | Push heads; pop min, push its next node | $O(n \log k) / O(k)$ |

| 43 | **Copy List with Random Ptr** | HashMap / Interleave | Two-pass with `old -> new` map; or weave copies in | $O(n) / O(1)$ |

### 💡 Implementation Secrets
* **The Dummy Node:** Whenever you might need to change the `head` of the list (like in Merge or Reverse), start with `ListNode dummy = new ListNode(0); dummy.next = head;`. It saves you from writing multiple `if (head == null)` checks.
* **Fast/Slow (Floyd's Cycle):** If the fast pointer (`fast.next.next`) ever equals the slow pointer (`slow.next`), there is a cycle. To find the *start* of the cycle, reset `slow` to the head and move both one step at a time; they will meet at the entry point.
* **In-Place Reversal:** Practice the 3-pointer swap (`prev`, `curr`, `next`) until you can do it from memory; it is a building block for many Hard-level problems.
