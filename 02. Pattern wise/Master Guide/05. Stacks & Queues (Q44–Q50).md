# 🥞 Section 5: Stacks & Queues (Q44–Q50)

#### **Core Concept:** Monotonic stacks are the "secret weapon" for solving 'next greater/smaller element' problems in linear $O(n)$ time. 
#### Deques are essential for sliding window maximums, and Queues are the engine behind Breadth-First Search (BFS). 
#### **Key Question:** "Is there a monotone ordering (increasing or decreasing) I can exploit here?"

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 44 | **Valid Parentheses** | Stack | Push opening bracket; on closing, pop and match | $O(n) / O(n)$ |
| 45 | **Min Stack** | Two Stacks / Pairs | Second stack (or paired value) tracks running minimum | $O(1)$ all ops / $O(n)$ |
| 46 | **Daily Temperatures** | Monotonic Dec. Stack | Store indices; pop when a warmer day is found | $O(n) / O(n)$ |
| 47 | **Eval Reverse Polish Notation** | Stack | Numbers $\rightarrow$ push; Operator $\rightarrow$ pop two, compute, push | $O(n) / O(n)$ |
| 48 | **Next Greater Element I** | Monotonic Stack + Map | Process `nums2` with stack; map each num to its next greater | $O(m+n) / O(n)$ |
| 49 | **Largest Rectangle in Histogram** | Monotonic Stack | Pop on shorter bar; compute area with `height * width` | $O(n) / O(n)$ |
| 50 | **Decode String** | Two Stacks | One stack for counts, one for strings; pop on `']'` | $O(n) / O(n)$ |

---

### 💡 Implementation Secrets
* **Why Indices?** In monotonic stack problems (like Q46 or Q49), storing the **index** in the stack is almost always better than storing the value. Indices allow you to calculate distances/widths ($i - \text{stack.peek()}$) while still giving you access to the values.
* **The "Dummy" Bar:** For "Largest Rectangle in Histogram," adding a bar with height `0` at the very end of your input ensures that the stack is completely emptied and all possible rectangles are calculated.
* **Stack for Recursion:** Remember that any recursive problem can be solved iteratively using a Stack. If you're hitting a `StackOverflowError`, it's time to switch to an explicit `java.util.Deque` (used as a Stack).
