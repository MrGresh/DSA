# 🧩 Section 10: Tries & Backtracking (Q99–Q100)

#### **Core Concept:** Tries (Prefix Trees) are the ultimate optimization for prefix-based operations, outperforming HashMaps by storing shared prefixes. 
#### Backtracking is essentially a DFS with a critical "undo" step: **Choose → Explore → Unchoose**. To avoid Time Limit Exceeded (TLE) errors, always look for ways to "prune" your search tree early.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 99 | **Word Search II** | Trie + DFS Backtracking | Build trie from word list; DFS the grid with trie pruning | $O(m \cdot n \cdot 3^L) / O(W \cdot L)$ |
| 100 | **Design Add and Search Words** | Trie + DFS on '.' | Standard trie insert; '.' wildcards trigger recursive DFS on all children | $O(m)$ ins, $O(26^m)$ worst / $O(W \cdot L)$ |

---

### 💡 Implementation Secrets
* **Trie Nodes:** In Java, a Trie node is typically a simple class with an array `TrieNode[26] children` and a boolean `isEndOfWord`.
* **Pruning in Backtracking:** In Word Search II, if the current character on the grid does not exist as a child in your current Trie node, stop exploring that path immediately. This is the difference between an $O(n^2)$ and an $O(\text{exponential})$ runtime.
* **The Wildcard Logic:** For the "Add and Search Words" problem, the `.` character is handled by iterating through all non-null indices in the `children` array and recursing. If any return `true`, the path is valid.
