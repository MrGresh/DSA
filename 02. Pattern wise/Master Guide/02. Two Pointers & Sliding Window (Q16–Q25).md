# ↔️ Section 2: Two Pointers & Sliding Window (Q16–Q25)

#### **Core Concept:** Two Pointers typically reduces $O(n^2)$ to $O(n)$ on sorted arrays.
#### Sliding Window maintains a `[left, right]` range—expand right always, shrink left only when the window becomes invalid.
#### The window represents your "candidate solution" at any given point.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 16 | **Trapping Rain Water** | Two Pointers / Stack | `left_max` & `right_max`; or two-pointer $O(1)$ space | $O(n) / O(1)$ |
| 17 | **Merge Sorted Arrays In-Place** | Two Pointers (Back) | Fill from back to front to avoid shifting elements | $O(m+n) / O(1)$ |
| 18 | **Move Zeroes** | Two Pointers | Slow pointer tracks next non-zero position | $O(n) / O(1)$ |
| 19 | **Squares of Sorted Array** | Two Pointers (Ends) | Compare `abs(left)` vs `abs(right)`, fill result from back | $O(n) / O(n)$ |
| 20 | **Subarray Sum Equals K** | Prefix Sum + HashMap | Map `prefixSum -> count`; check if `sum - k` exists | $O(n) / O(n)$ |
| 21 | **Sliding Window Maximum** | Monotonic Deque | Deque stores indices in decreasing value order | $O(n) / O(k)$ |
| 22 | **Permutation in String** | Sliding Window + Freq Map | Fixed window of size `len(p)`, compare char counts | $O(n) / O(1)$ |
| 23 | **Minimum Size Subarray Sum** | Sliding Window | Shrink left when `sum >= target`; track min length | $O(n) / O(1)$ |
| 24 | **Max Consecutive Ones III** | Sliding Window | Allow flipping `k` zeros; track zero count in window | $O(n) / O(1)$ |
| 25 | **Longest Subarray with Ones after Deletion** | Sliding Window | Maintain at most 1 zero in the window | $O(n) / O(1)$ |

---

### 💡 Implementation Secrets
* **The "Shrink" Logic:** For variable-sized sliding windows (like Q23), the `while` loop that shrinks the left pointer is where you usually update your global minimum.
* **Monotonic Deque:** For "Sliding Window Maximum," the deque helps you find the maximum in $O(1)$ by keeping elements in a specific order and popping those that are no longer useful.
* **Why Fill from Back?** In problems like "Merge Sorted Arrays," filling from the back prevents you from overwriting values you haven't processed yet, saving you from needing extra space.
