# 🕸️ Section 8: Graphs (Q73–Q85)

#### **Core Concept:** Almost every graph problem fits into one of these categories: 
#### **BFS** (shortest path), **DFS** (connectivity/cycle detection), **Union-Find** (dynamic components), 
#### **Dijkstra** (weighted shortest path), or **Topological Sort** (dependency ordering). 
#### Identify the category before you write a single line of code.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 73 | **Number of Islands** | DFS/BFS Flood Fill | Visit island cells, mark visited, count components | $O(m \cdot n) / O(m \cdot n)$ |

### 💡 Implementation Secrets
* **The "3-Color" DFS:** When detecting cycles in a directed graph, use an array to track states: `0` (unvisited), `1` (currently in recursion stack), and `2` (completely processed). If you hit a `1`, you found a cycle.
* **Multi-source BFS:** Instead of running BFS multiple times, push all starting points (like all rotten oranges or all gates) into the queue at once. This ensures you find the global shortest path in a single pass.
* **Kahn's Algorithm:** This is just BFS for Topological Sort. The "trigger" is a node's **indegree** hitting zero.
* **Union-Find (DSU):** Always implement **Path Compression** and **Union by Rank/Size**. This keeps the operations nearly $O(1)$.
