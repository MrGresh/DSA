# 🔍 Section 3: Binary Search (Q26–Q33)

#### **Core Concept:** Binary Search isn't just for sorted arrays.
#### Any **monotone predicate** $f(x)$ can be binary searched.
#### "Search on the Answer" is the key insight: if you can check "is $X$ feasible?" in $O(n)$, you can binary search over the range of possible $X$ values.

---

### 📋 Essential Problem Set

| # | Question | Key Concept | Approach | Complexity (T/S) |
| :-- | :--- | :--- | :--- | :--- |
| 26 | **Binary Search (Classic)** | Binary Search | `lo=0, hi=n-1`; `mid = lo + (hi-lo)/2` | $O(\log n) / O(1)$ |
| 27 | **First and Last Position** | Binary Search x2 | Two binary searches for left and right bound | $O(\log n) / O(1)$ |
| 28 | **Search a 2D Matrix** | BS on Flat Index | Treat as 1D; `row = mid / cols`, `col = mid % cols` | $O(\log(m \cdot n)) / O(1)$ |
| 29 | **Koko Eating Bananas** | BS on Answer | Search $k$ in `[1, max(piles)]`; check feasibility | $O(n \log M) / O(1)$ |
| 30 | **Min in Rotated Sorted II** | Modified BS | When `mid == right`, do `right--` to skip duplicates | $O(\log n)$ avg / $O(1)$ |
| 31 | **Capacity to Ship in D Days** | BS on Answer | `lo = max(weights)`, `hi = sum`; check if days $\le D$ | $O(n \log S) / O(1)$ |
| 32 | **Split Array Largest Sum** | BS on Answer | Identical logic to the "Ship Packages" pattern | $O(n \log S) / O(1)$ |
| 33 | **Median of Two Sorted Arrays** | BS on Partition | Partition both; check `max_left <= min_right` | $O(\log(\min(m,n))) / O(1)$ |

### 💡 Implementation Secrets
* **The "Mid" Calculation:** Always use `mid = lo + (hi - lo) / 2` instead of `(lo + hi) / 2` to prevent **integer overflow** in Java/C++.
* **Search on Answer:** If the problem asks for a "minimum possible maximum" or "maximum possible minimum," it is almost certainly a Binary Search on Answer problem.
* **The 2D Trick:** Mapping a 2D index $(r, c)$ to a 1D index $i$ is done via $i = r \cdot \text{cols} + c$. To go backwards from $i$ to $(r, c)$, use $r = i / \text{cols}$ and $c = i \% \text{cols}$.
