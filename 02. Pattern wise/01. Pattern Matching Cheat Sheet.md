A quick-reference guide to help identify the right data structures and algorithms based on common problem descriptions.

| If the problem says... | Your first instinct should be... |
| :--- | :--- |
| **Top K** / **Kth** largest or smallest | 🪵 **Min-Heap** of size K |
| **Shortest path** (unweighted graph) | 🗺️ **BFS** |
| All **combinations** / **subsets** / **permutations** | 🔄 **Backtracking** (DFS + undo) |
| **Subarray** / **substring** with a constraint | 🪟 **Sliding Window** |
| **Sorted array** + find a pair | 👈 **Two Pointers** 👉 |
| **Max/min** of something over choices | 📈 **Dynamic Programming** |
| **Word prefix** / **autocomplete** / **search** | 🌲 **Trie** |
| **Parentheses** / **expression evaluation** | 🥞 **Stack** |
| **Connected components** / **union queries** | 🕸️ **Union-Find (DSU)** |
| **Count ways** / **how many paths** | 🔢 **DP** (usually 1D or 2D) |
| **Overlapping intervals** / **merge intervals** | 🗓️ **Sort by start + Greedy** |
| **Tree structure** with **ordered property** | 🌳 **BST algorithms** |
| **Repeated subproblems** in recursion | 💾 **Memoization / Tabulation** |
