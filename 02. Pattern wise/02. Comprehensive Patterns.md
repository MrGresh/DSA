# 📖 DSA Patterns: The Comprehensive Guide

> **Purpose:** This guide serves as a high-level taxonomy of all major Data Structure and Algorithm patterns. Use this to quickly identify which "family" of algorithms a problem belongs to during your preparation.

---

## 1. Array & String Patterns
* **Two Pointers:**
    * *Converging:* Pointers start at opposite ends (e.g., Two Sum in sorted array).
    * *Constant Distance:* Pointers move together (e.g., Remove duplicates).
    * *Different Arrays:* One pointer per array (e.g., Merging sorted arrays).
* **Sliding Window:**
    * *Fixed-size:* Track a specific subarray length (e.g., Max sum of K elements).
    * *Variable-size:* Expand/shrink based on conditions (e.g., Smallest subarray with sum S).
* **Prefix Sum / Difference Array:** Precomputing cumulative values to answer range queries in $O(1)$.
* **Kadane’s Algorithm:** Linear time approach to find the maximum sum subarray.
* **Boyer-Moore Voting:** Finding the majority element ($>N/2$ times) in $O(N)$ time and $O(1)$ space.
* **Dutch National Flag:** Sorting an array of three distinct elements (0s, 1s, 2s) in a single pass.
* **Cyclic Sort:** $O(n)$ sorting for arrays containing numbers in a known range (1 to $n$).

## 2. Linked List Patterns
* **Fast & Slow Pointers (Tortoise & Hare):** Detecting cycles or finding the middle of a list.
* **In-place Reversal:** Reversing nodes without using extra memory.
* **Reverse in K-Groups:** Reversing specific segments of a list iteratively.
* **Intersection / Cycle Detection:** Finding where two lists meet or if a loop exists.
* **Dummy Node Technique:** Using a placeholder head node to simplify edge cases (e.g., deleting the first node).

## 3. Stack & Queue Patterns
* **Monotonic Stack:** Maintaining elements in increasing/decreasing order to find the "Next Greater/Smaller Element."
* **Monotonic Queue:** Optimized for "Sliding Window Maximum" problems.
* **Parentheses Matching:** Using a stack to validate nested structures and expressions.
* **Breadth-First Search (BFS):** Using a queue for level-by-level processing.

## 4. Tree & Graph Patterns
* **DFS (Depth-First Search):** Traversing branches via recursion (Pre-order, In-order, Post-order).
* **BFS (Breadth-First Search):** Level-order traversal for shortest paths in unweighted graphs.
* **Topological Sort:** Ordering nodes in a Directed Acyclic Graph (DAG) (e.g., task scheduling).
* **Union Find (DSU):** Efficiently handling connected components and cycle detection.
* **Shortest Path:**
    * *Dijkstra:* Weighted graphs (no negative edges).
    * *Bellman-Ford:* Handles negative weights and detects negative cycles.
    * *Floyd-Warshall:* All-pairs shortest path.
* **Trie (Prefix Tree):** Optimized for string searches and prefix matching.
* **Segment/Fenwick Trees:** Advanced structures for range sum/update queries in $O(\log N)$.

## 5. Heap / Priority Queue Patterns
* **Top 'K' Elements:** Finding the largest/smallest K elements in a stream.
* **K-way Merge:** Merging K sorted arrays/lists efficiently.
* **Two Heaps:** Maintaining a Min-Heap and Max-Heap to find a rolling median.

## 6. Searching & Sorting Patterns
* **Two Pointer after Sorting:** Sort the array, then use two pointers to efficiently find pairs/triplets.
* **Lower Bound / Upper Bound:** Find the first or last valid position using binary search.
* **Search in Rotated Sorted Array:** Modified binary search by identifying the sorted half.
* **Counting Sort:** Use frequency arrays when elements have a limited range.
* **Bucket Sort:** Distribute elements into buckets to sort efficiently for uniform data.
* **Custom Comparator Sorting:** Sort based on problem-specific rules instead of natural order.
* **Partial Sorting:** Sort only the required portion to optimize performance.
* **Merge-based Logic:** Use merge sort ideas to count inversions or solve range problems.
* **Selection via Sorting:** Sort elements when order matters more than exact positions.
* **Search in Infinite / Unknown Size Array:** Expand the search space exponentially ($2^k$), then apply binary search.

## 7. Recursion & Backtracking Patterns
* **Subsets / Permutations:** Generating all possible combinations of data.
* **Constraint Satisfaction:** Problems like Sudoku or N-Queens where you explore and prune invalid paths.

## 8. Dynamic Programming (DP) Patterns
* **0/1 Knapsack:** Pick or skip items to maximize value under a weight constraint.
* **LCS / LIS:** Identifying Longest Common Subsequence or Longest Increasing Subsequence.
* **DP on Trees:** Calculating values (like diameter or max path sum) based on child node results.
* **Bitmask DP:** Solving NP-hard problems (like TSP) for small input sizes using bit manipulation.

## 9. Bit Manipulation Patterns
* **Bit Masking:** Representing sets or states as integers.
* **XOR Properties:** $A \oplus A = 0$ and $A \oplus 0 = A$ (useful for finding unique elements).
* **Brian Kernighan’s Algorithm:** Efficiently counting set bits in an integer by clearing the least significant bit.
