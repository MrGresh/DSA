## 🛠 Useful Patterns to Remember

### 1. Monotonic Stack
Used for "Next Greater/Smaller" problems. You maintain a stack in either strictly increasing or decreasing order to find the nearest element that satisfies a condition in $O(n)$.


### 2. Expression Parsing (Shunting-Yard)
Essential for Infix, Prefix, and Postfix conversions. Remember:
* **Infix to Postfix:** Use a stack for operators based on precedence.
* **Evaluation:** Use a stack for operands.

### 3. Min/Max Stack ($O(1)$)
A common interview trick. Either use a secondary "min stack" or use a mathematical formula to store the previous minimum within the current value:
$NewValue = 2 \times CurrentElement - CurrentMin$

### 4. Recursive Reversal
To reverse a stack without extra space, use the function call stack. Use a helper function `insertAtBottom` to place elements after popping the entire stack.

### 5. Circular Queue
To implement a queue in a fixed-size array, use the modulo operator:
$rear = (rear + 1) \% size$

### 6. Two Stacks = One Queue
By using two stacks (one for `push`, one for `pop`), you can simulate a FIFO structure. Amortized time complexity for operations remains $O(1)$.
