## 🛠 Useful Patterns to Remember

### 1. BFS vs. DFS
* **BFS:** Shortest path in unweighted graphs. Uses a **Queue**.
* **DFS:** Connectivity, cycle detection, and deep exploration. Uses **Recursion/Stack**.

### 2. The shortest Path "Hierarchy"
* **Unweighted:** BFS ($O(V+E)$)
* **Weighted (Positive):** Dijkstra ($O(E \log V)$)
* **Weighted (Negative edges):** Bellman-Ford ($O(VE)$)
* **All-Pairs Shortest Path:** Floyd-Warshall ($O(V^3)$)

### 3. Topological Sort (Kahn’s Algorithm)
Used for Directed Acyclic Graphs (DAGs). It’s the "ordering" of nodes such that for every directed edge $u \to v$, node $u$ comes before $v$. Essential for dependency/scheduling problems.

### 4. Disjoint Set Union (DSU)
Provides two operations: `Find` (which set an element belongs to) and `Union` (merge two sets). Vital for Kruskal's MST and checking connectivity in dynamic graphs.

### 5. Multi-Source BFS
For problems where "infection" or "distance" starts from multiple points simultaneously (like Rotten Oranges). Push all starting nodes into the queue at once before beginning the BFS.

### 6. Strongly Connected Components (SCC)
A directed graph is strongly connected if there is a path between every pair of vertices. 
* **Kosaraju's:** 2 DFS passes.
* **Tarjan's:** 1 DFS pass with low-link values.
