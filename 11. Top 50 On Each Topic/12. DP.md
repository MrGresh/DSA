## 🛠 Useful Patterns to Remember

### 1. Memoization vs. Tabulation
* **Memoization (Top-Down):** Recursion + Hashmap/Array. Solves only the subproblems required.
* **Tabulation (Bottom-Up):** Iterative. Fills a table from smallest to largest subproblem. Better for space optimization (e.g., $O(1)$ space Fibonacci).

### 2. State Definition
The most critical step in DP. Ask: "What information do I need at index $i$ to make a decision?"
* Examples: $DP[i][j]$ for substrings, $DP[i][weight]$ for Knapsack.

### 3. LCS and LIS Variants
Mastering **Longest Common Subsequence** and **Longest Increasing Subsequence** unlocks many other problems like Edit Distance, Rod Cutting, and Maximum Sum Increasing Subsequence.

### 4. String DP (2D Table)
For string comparison/alignment, use a table where rows represent string A and columns represent string B. $DP[i][j]$ usually relates characters $A[i-1]$ and $B[j-1]$.


### 5. Partition DP
Used when you need to split an array or string into multiple segments optimally (e.g., Matrix Chain Multiplication or Palindrome Partitioning). The recurrence usually looks like:
$DP[i][j] = \min/\max \{ DP[i][k] + DP[k+1][j] + cost \}$ for all $k$ from $i$ to $j-1$.

### 6. Space Optimization Trick
If the current state $DP[i]$ only depends on the previous row $DP[i-1]$, you can often reduce the space complexity from $O(N^2)$ to $O(N)$ by using only two rows or a single row (in-place update).
