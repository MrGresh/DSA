## 🛠 Useful Patterns to Remember

### 1. Monotonic Deque
The most powerful pattern for "Sliding Window" problems. 
* To find the **Maximum** in a window: Maintain the Deque in **decreasing order**. If a new element is larger than the back, pop from the back until it's not.
* To find the **Minimum**: Maintain the Deque in **increasing order**.


### 2. 0-1 BFS (Optimization)
When graph edges have only two weights (0 and 1), standard Dijkstra is overkill. 
* Use a Deque: If the weight is **0**, `push_front`. If the weight is **1**, `push_back`. This keeps the Deque sorted by distance without a Priority Queue.

### 3. Sliding Window Maximum
This is the "Hello World" of Deque problems. The Deque stores **indices** instead of values. Indices are popped from the front if they are outside the current window range $(i - k)$.

### 4. Space Complexity Advantage
A Deque allows you to implement both LIFO (Stack) and FIFO (Queue) logic. In many languages (like Python's `collections.deque` or C++'s `std::deque`), it is implemented as a sequence of individually allocated blocks, making it highly efficient for large datasets compared to a vector/array that requires contiguous memory.

### 5. Bitonic & Zigzag Traversal
Deques are excellent for building bitonic sequences or printing trees in zigzag fashion, where you need to toggle the end from which you extract and insert elements at each level.
