## 🛠 Useful Patterns to Remember

### 1. The Recursive Leap of Faith
Most tree problems follow a standard template:
1. Handle the `base case` (usually `if root is NULL`).
2. Solve the `left` subtree.
3. Solve the `right` subtree.
4. `Combine` the results at the current node.

### 2. Level Order Traversal (BFS)
Using a **Queue**, you can traverse a tree level by level. This is essential for problems like "Zigzag," "Vertical Traversal," or "Connecting Nodes."


### 3. Lowest Common Ancestor (LCA)
A fundamental building block. Once you find the LCA of two nodes, you can calculate the distance between them:
$Dist(u, v) = Dist(root, u) + Dist(root, v) - 2 \times Dist(root, LCA)$

### 4. Morris Traversal
An advanced technique to perform **Inorder Traversal** with $O(1)$ space by utilizing the "null" pointers in the leaf nodes to point back to the ancestors (Threaded Binary Tree).

### 5. BST Property Validation
To check if a tree is a BST, don't just check if `left < root < right`. You must maintain a `min` and `max` range that each node must fall within as you recurse down.


### 6. Tree DP (Maximum Sum Path)
For problems like "Maximum Sum Path," each node returns the maximum path it can provide to its parent, while simultaneously updating a global maximum that includes both its children.
