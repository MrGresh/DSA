# ⚡ Bit Magic Mastering Roadmap

A collection of essential bitwise techniques and problems to master low-level data manipulation, parity, and binary logic.

## 🟢 Essential Techniques & Problems

1. [**Compute XOR from 1 to n (Direct Method)**](https://www.geeksforgeeks.org/calculate-xor-1-n/)
   * **Pattern:** Observe the cycle of 4:
     - If `n % 4 == 0`, result is `n`
     - If `n % 4 == 1`, result is `1`
     - If `n % 4 == 2`, result is `n + 1`
     - If `n % 4 == 3`, result is `0`

2. [**Count x ≤ n such that (n + x) == (n ^ x)**](https://www.geeksforgeeks.org/count-number-of-x-less-than-or-equal-to-n-such-that-n-x-nx/)
   * **The Logic:** This holds true only if there are no "carries" during addition.
   * **The Constraint:** `(n & x) == 0`
   * **The Result:** $2^{k}$, where $k$ is the number of unset bits (zeros) in $n$.

3. [**How to Know if a Number is a Power of 2?**](https://www.geeksforgeeks.org/program-to-find-whether-a-given-number-is-power-of-2/)
   * **The One-Liner:** `return n && !(n & (n - 1));`
   

4. [**Find XOR of All Subsets of a Set**](https://www.geeksforgeeks.org/xor-of-all-subset-xors-of-an-array/)
   * **Result:** Always `0` if the set has more than 1 element, because every element appears an even number of times ($2^{n-1}$) across all subsets.

5. [**Leading, Trailing Zeroes, and Number of 1s**](https://www.geeksforgeeks.org/builtin-functions-gcc-compiler/)
   * Use GCC Built-ins for $O(1)$ speed:
     - `__builtin_clz(x)`: Count Leading Zeroes
     - `__builtin_ctz(x)`: Count Trailing Zeroes
     - `__builtin_popcount(x)`: Count set bits (1s)

6. [**Convert Binary Code Directly into an Integer**](https://www.geeksforgeeks.org/stdstoi-function-in-cpp/)
   * **C++ Helper:** `stoi(binary_string, 0, 2);` or `bitset<32>(str).to_ulong();`

7. [**The Quickest Way to Swap Two Numbers**](https://www.geeksforgeeks.org/swap-two-numbers-without-using-temporary-variable/)
   * **XOR Swap:**
     ```cpp
     a ^= b;
     b ^= a;
     a ^= b;
     ```

8. [**Simple Approach to Flip the Bits of a Number**](https://www.geeksforgeeks.org/flipping-the-bits-of-a-number-at-given-position/)
   * **Logic:** To flip bits up to the MSB, use a mask: `n ^ ((1 << total_bits) - 1)`

9. [**Finding the Most Significant Set Bit (MSB)**](https://www.geeksforgeeks.org/find-significant-set-bit-number/)
   * **Shortcut:** `1 << (31 - __builtin_clz(n))` for 32-bit integers.

10. [**Check if a Number has Bits in an Alternate Pattern**](https://www.geeksforgeeks.org/check-if-a-number-has-bits-in-alternate-pattern/)
    * **Trick:** 1. Let `temp = n ^ (n >> 1)`
      2. If $n$ was alternating (e.g., `1010`), `temp` will be all 1s (`1111`).
      3. Check: `((temp + 1) & temp) == 0`

## 🛠 Useful Bitwise Properties

### 1. The `n & (n-1)` Trick
Clears the **rightmost set bit**. Essential for:
* **Hamming Weight:** Counting total set bits.
* **Power of 2 check:** Seeing if only one bit was set.


### 2. The `n & -n` Trick (LowBit)
Extracts the **lowest set bit** as a value.
* For $12$ (`1100`), it returns $4$ (`0100`).
* Fundamental for **Fenwick Trees (Binary Indexed Trees)**.

### 3. XOR Identity Rules
* $A \oplus A = 0$
* $A \oplus 0 = A$
* If $A \oplus B = C$, then $A \oplus C = B$

### 4. Bit Masking 101
* **Check $i$-th bit:** `(n >> i) & 1`
* **Set $i$-th bit:** `n | (1 << i)`
* **Clear $i$-th bit:** `n & ~(1 << i)`
* **Toggle $i$-th bit:** `n ^ (1 << i)`

### 5. Subset Generation (Bitmasking)
To find all subsets of a set of size $n$, iterate from $0$ to $(2^n - 1)$. The binary of each loop index acts as a selector for the elements.
