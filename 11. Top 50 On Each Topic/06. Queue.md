## 🛠 Useful Patterns to Remember

### 1. BFS (Breadth-First Search)
The Queue is the core data structure for BFS. It is used to find the **shortest path** in unweighted graphs or grids. 
* **Key Idea:** Level-order traversal where you visit all neighbors at distance 1 before distance 2.

### 2. Multi-Source BFS
Used in problems like "Rotten Oranges." Instead of starting from one node, push all starting points (rotten oranges) into the queue initially to find the global minimum time.

### 3. Sliding Window (Deque)
For "Max/Min in every window of size K" or "First Negative integer," a **Double-Ended Queue (Deque)** allows you to push and pop from both ends in $O(1)$, maintaining only the "useful" elements for the current window.


### 4. Monotonic Queue
Similar to a monotonic stack, this queue keeps elements in a specific order (increasing or decreasing). It helps in finding the maximum or minimum in a sliding window in linear time.

### 5. Circular Queue (Space Optimization)
To avoid shifting elements when popping from an array-based queue, use two pointers (`front` and `rear`) and the modulo operator:
`rear = (rear + 1) % capacity`


### 6. Queue with Two Stacks
To simulate a Queue using Stacks:
* **Push:** Add to `stack1`.
* **Pop:** If `stack2` is empty, move everything from `stack1` to `stack2`. Pop from `stack2`.
