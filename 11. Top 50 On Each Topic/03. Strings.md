## 🛠 Useful Patterns to Remember

### 1. Sliding Window (Frequency Maps)
Used for substring problems. Maintain a window $[i, j]$ and a hashmap/frequency array of size 256 to track character counts.

### 2. Two Pointers (Expand from Center)
The most efficient way to find palindromic substrings. Pick a center (or two for even lengths) and expand outwards.


### 3. Hashing (Rabin-Karp)
Treat the string as a number in base $b$ (usually 26 or 31) and use a rolling hash to compare patterns in $O(1)$ after precomputation.

### 4. KMP Algorithm (LPS Array)
Precompute the "Longest Proper Prefix which is also a Suffix" (LPS) to avoid redundant comparisons during pattern matching.

### 5. String DP
Many "Hard" string problems are solved by filling a 2D table where $DP[i][j]$ represents a property of the substring $S[i...j]$ (e.g., LCS, LPS, or Edit Distance).

### 6. String Tries
For prefix-based problems or finding multiple patterns in a large body of text, a Trie (Prefix Tree) is the go-to data structure.
