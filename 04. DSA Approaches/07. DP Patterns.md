**Dynamic Programming (DP) in DSA revolves around recognizing recurring problem structures and applying reusable templates. Instead of memorizing solutions, mastering these patterns helps you solve unseen problems efficiently.**  

---

## 🔑 Core DP Patterns in DSA
Here are the most widely recognized DP templates you’ll encounter in interviews and competitive programming:

### 1. **1D DP (Linear Progression)**
- Problems: Fibonacci, Climbing Stairs, Minimum Cost Climbing Stairs  
- Idea: Build solutions iteratively using previous states.  
- Transition: `dp[i] = dp[i-1] + dp[i-2]`

### 2. **2D DP (Grid/Matrix Problems)**
- Problems: Unique Paths, Minimum Path Sum, Maximum Gold Collection  
- Idea: Each cell depends on top/left neighbors.  
- Transition: `dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`

### 3. **Knapsack Variants**
- **0/1 Knapsack**: Choose or skip items.  
- **Unbounded Knapsack**: Items can be reused.  
- Problems: Coin Change, Rod Cutting, Subset Sum.  
- Transition: `dp[i][w] = max(dp[i-1][w], value[i] + dp[i-1][w-weight[i]])`

### 4. **Longest Sequence Problems**
- **Longest Increasing Subsequence (LIS)**  
- **Longest Common Subsequence (LCS)**  
- Transition:  
  - LIS: `dp[i] = max(dp[j] + 1)` for all `j < i` with `arr[j] < arr[i]`  
  - LCS: `dp[i][j] = 1 + dp[i-1][j-1]` if `s1[i] == s2[j]`

### 5. **Partitioning & Subset Problems**
- Problems: Equal Subset Sum Partition, Palindrome Partitioning.  
- Idea: Break problem into smaller partitions and solve recursively with memoization.

### 6. **Interval DP**
- Problems: Matrix Chain Multiplication, Burst Balloons.  
- Idea: Solve subproblems over ranges/intervals.  
- Transition: `dp[i][j] = min(dp[i][k] + dp[k+1][j] + cost(i,j))`

### 7. **Digit DP**
- Problems: Count numbers with certain properties (e.g., no consecutive digits).  
- Idea: State defined by position, tightness, and constraints.

### 8. **Bitmask DP**
- Problems: Traveling Salesman Problem (TSP), Assignments.  
- Idea: Use bitmask to represent subsets.  
- Transition: `dp[mask][i] = min(dp[mask ^ (1<<i)][j] + dist[j][i])`

### 9. **DP on Trees**
- Problems: Diameter of Tree, Maximum Independent Set.  
- Idea: Use DFS + DP to compute values for subtrees.

### 10. **DP on Graphs**
- Problems: Shortest Paths (Bellman-Ford, Floyd-Warshall).  
- Idea: Iteratively relax edges or compute over adjacency matrix.

## 📊 Comparison Table of Key DP Patterns

| **Pattern**            | **Common Problems**                  | **State Definition**            | **Complexity** |
|-------------------------|--------------------------------------|---------------------------------|----------------|
| 1D DP                  | Fibonacci, Climbing Stairs           | Index position                  | O(n)           |
| 2D DP                  | Grid paths, Min Path Sum             | Cell coordinates (i, j)         | O(n·m)         |
| Knapsack Variants       | Coin Change, Subset Sum              | Item index + weight/capacity    | O(n·W)         |
| Longest Sequence        | LIS, LCS                            | Index + subsequence length      | O(n²) / O(n·m) |
| Interval DP             | Matrix Chain Multiplication          | Range (i, j)                    | O(n³)          |
| Bitmask DP              | TSP, Assignments                    | Subset mask + last node         | O(n·2ⁿ)        |
| DP on Trees             | Diameter, Independent Set            | Node + subtree states           | O(n)           |

## 🚀 How to Approach Any DP Problem
1. **Identify overlapping subproblems** (recursion with repeated calls).  
2. **Define states clearly** (indices, sums, masks, etc.).  
3. **Write recurrence relation** (transition formula).  
4. **Choose memoization or tabulation** depending on clarity and constraints.  
5. **Optimize space** (rolling arrays, bitmasks).
