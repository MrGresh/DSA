# 🗺️ The Ultimate Subarray Strategy Guide

A systematic framework for choosing the right algorithm for any subarray problem based on constraints and goals.

---

## 🚀 1. The Core Decision Matrix

Use this table to immediately identify the required pattern based on the array contents and the goal.

| Goal | Array Content | Pattern | Complexity |
| :--- | :--- | :--- | :--- |
| **Sum == K** | Any (±) | **Prefix Sum + HashMap** | $O(N)$ Time, $O(N)$ Space |
| **Sum ( inequalities )** | Positive Only | **Sliding Window** | $O(N)$ Time, $O(1)$ Space |
| **Sum ( inequalities )** | Any (±) | **Prefix Sum + Monotonic Stack** | $O(N)$ Time, $O(N)$ Space |
| **Max / Min Sum** | Any (±) | **Kadane’s Algorithm** | $O(N)$ Time, $O(1)$ Space |
| **Max / Min Product** | Any (±) | **Modified Kadane’s** | $O(N)$ Time, $O(1)$ Space |
| **Range Max / Min** | Any (±) | **Sliding Window + Deque** | $O(N)$ Time, $O(K)$ Space |

## 🧠 2. Deep Dive into Patterns

### A. Sliding Window (Two Pointers)
* **Rule of Thumb:** Use when the condition is **monotonic** (moving the right pointer only increases the sum/count).
* **Why it's the "Interview Favorite":** It achieves $O(1)$ space. Always suggest this first if the array is all positives.
* **Problems:** *Max Consecutive Ones III*, *Minimum Size Subarray Sum*.

### B. Prefix Sum + HashMap
* **Rule of Thumb:** Use for **Exact Equality** ($Sum == K$).
* **The Logic:** "Remembers" the past. If $CurrentSum - Target = X$, and $X$ was seen at index $i$, then the subarray from $i+1$ to $current$ equals the target.
* **Longest vs Shortest:**
    * **Longest:** Only store the *first* occurrence of a sum.
    * **Shortest:** Update the map with the *latest* occurrence.
* **Problems:** *Subarray Sum Equals K*, *Contiguous Array (0s and 1s)*.

### C. Prefix Sum + Monotonic Stack
* **Rule of Thumb:** The "Pro" version for inequalities with **negative numbers**.
* **The Logic:** When adding an element can make the sum go down (negatives), sliding window fails. A monotonic stack helps find the widest "gap" where $Prefix[j] - Prefix[i] \geq K$.
* **Problems:** *Longest Well-Performing Interval*, *Shortest Subarray with Sum at Least K*.

### D. Kadane’s Algorithm (DP)
* **Rule of Thumb:** Use for finding the **Global Maximum/Minimum** sum.
* **The Logic:** Local optimization. At each index, decide: "Should I start fresh here, or continue the previous subarray?"
* **Formula:** `currMax = max(arr[i], currMax + arr[i])`
* **Problems:** *Maximum Subarray*, *Maximum Product Subarray*.

## 🛠️ 3. The "Cheat Sheet" Rules

### 1. The Monotonicity Rule
* **Positives Only:** Prefix sum only goes up. Sliding window works.
* **Negatives Present:** Prefix sum fluctuates. Sliding window **fails**. Use HashMap or Stack.

### 2. The Equality Exception
* Even if the array is all positives, if the goal is **Sum == K**, use a **HashMap**. Sliding window is better for inequalities ($<, >$).

### 3. The "At Most K" Trick
* To find **Exactly K** (e.g., exactly K distinct integers), calculate:
    `countAtMost(K) - countAtMost(K - 1)`.

## ⚠️ Common Interview Pitfalls

* **The Zero in Product:** In *Max Product Subarray*, a zero resets everything. Handle it by resetting Kadane variables or doing a two-pass (Left-to-Right and Right-to-Left).
* **Space Optimization:** If you suggest a HashMap solution for an all-positive array, a good interviewer will ask: *"Can you do this in O(1) space?"* (The answer is Sliding Window).
* **Binary Search on Answer:** If you are asked for the "Minimum length $L$ such that..." and the check function is easy to write, consider Binary Search on the length $L$.

---

### 🚀 Strategy
**Prefix Sum + HashMap** for Online Assessments (OA) and **Sliding Window** for technical interviews. When starting a subarray problem, **always ask if the input contains negative numbers.** It changes your entire approach.
