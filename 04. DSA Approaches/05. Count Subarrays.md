# 📊 Subarray Counting Patterns

A guide to efficiently counting subarrays that satisfy specific conditions. For an $N = 10^5$ constraint, always aim for **O(N)** or **O(N log N)**.

---

## 1. Sliding Window (Two Pointers)
**Use when:** The array contains **non-negative** integers and the condition is **monotonic** (as the window grows, the condition remains true or moves closer to being false).

* **Logic:** For every fixed `right` pointer, if the window `[left, right]` is valid, then all windows `[left+1, right], [left+2, right] ... [right, right]` are also valid.
* **Formula:** `count += (right - left + 1)`
* **Common Problems:** * Subarrays with sum less than $K$.
    * Subarrays with at most $K$ distinct elements.

---

## 2. Prefix Sum + HashMap
**Use when:** The array contains **negative** numbers (non-monotonic) or you need an **exact** target (Sum $= K$).

* **Logic:** Store the frequency of `prefixSum` in a Map. If we are at `currentSum`, we look for `currentSum - target` in the Map.
* **Formula:** `count += map.getOrDefault(currentSum - target, 0)`
* **Common Problems:**
    * Subarray Sum Equals $K$.
    * Subarrays divisible by $K$ (use `prefixSum % K`).
    * Subarrays with equal number of 0s and 1s (replace 0 with -1).

---

## 3. The "At Most" Transformation
**Use when:** You are asked for **exactly K** of something, but the sliding window logic only works for "At Most."

* **Logic:** Counting "Exactly $K$" directly is often hard because the window doesn't shrink predictably.
* **Formula:** `Exactly(K) = AtMost(K) - AtMost(K - 1)`
* **Common Problems:**
    * Subarrays with exactly $K$ different integers.
    * Subarrays with exactly $K$ odd numbers.

---

## 4. Contribution to Sum (Monotonic Stack)
**Use when:** You need to count subarrays where a specific element `arr[i]` acts as the **Minimum** or **Maximum**.

* **Logic:** Find the nearest smaller/larger element to the left (`L`) and right (`R`).
* **Formula:** Subarrays where `arr[i]` is the min = `(i - L) * (R - i)`
* **Common Problems:**
    * Sum of Subarray Minimums.
    * Sum of Subarray Ranges.

---

## 💡 Quick Reference Table

| Condition | Pattern | Data Structure |
| :--- | :--- | :--- |
| **Sum < K (Positives)** | Sliding Window | None |
| **Sum = K (Negatives)** | Prefix Sum | HashMap |
| **Exactly K Distinct** | AtMost(K) - AtMost(K-1) | HashMap / Frequency Array |
| **Subarray Min/Max** | Contribution Technique | Monotonic Stack |

### 🚀 Interview Tip
During the **SDE-1 technical round**, if you see a subarray problem, immediately ask: *"Do we have negative numbers?"* - If **No**: Suggest Sliding Window ($O(N)$ time, $O(1)$ space).
- If **Yes**: Suggest Prefix Sum + HashMap ($O(N)$ time, $O(N)$ space).
This shows you are thinking about **space-time complexity trade-offs**, a key Amazon competency.
