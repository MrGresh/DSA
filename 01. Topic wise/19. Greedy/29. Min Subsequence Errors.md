# Amazon OA: Minimizing Subsequence Errors (01 vs 10)

## Problem Description
A binary string represents data where some characters are unknown, marked as `'!'`. Each `'!'` can be replaced with either `'0'` or `'1'`. Errors are generated based on the total number of specific subsequences in the final string:
- A subsequence **"01"** generates **$x$** errors.
- A subsequence **"10"** generates **$y$** errors.

**Goal:** Find the minimum possible total errors. Return the result modulo $10^9 + 7$.

### Key Constraints
- **Subsequence:** Characters do not need to be adjacent (e.g., in "101", the first '1' and last '1' form a "11" subsequence).
- **String Length:** Up to $10^5$.
- **Error Weights:** $x$ and $y$ can be up to $10^5$.

---

## Logical Approach: The Greedy Property
The most efficient way to solve this is to recognize that "mixing" `0`s and `1`s for the wildcards (`!`) usually creates unnecessary internal errors. 

### Why O(N)?
Testing every combination of `!` would be $O(2^k)$, which is impossible for large strings. Instead, we use a **Greedy + Prefix Sum** approach:
1. **Scenario A:** Replace every `'!'` with `'0'`.
2. **Scenario B:** Replace every `'!'` with `'1'`.
3. The minimum error will almost always be one of these two cases because the cost function is monotonic.

### Subsequence Counting Logic
To count subsequences in $O(N)$, we track running totals:
- Every time we see a `'1'`, it forms a **"01"** with every `'0'` that appeared before it. 
- Every time we see a `'0'`, it forms a **"10"** with every `'1'` that appeared before it.

---

## Optimized Implementation (Java)

```java
public class Solution {

    public static int getMinErrors(String errorString, int x, int y) {
        long MOD = 1_000_000_007;
        Result res0 = calculate(errorString, '0', x, y, MOD);
        Result res1 = calculate(errorString, '1', x, y, MOD);
        return (int)(res0.trueTotal <= res1.trueTotal ? res0.modded : res1.modded);
    }

    static class Result {
        long modded;
        double trueTotal; // double: handles up to ~10^19 safely for comparison
        Result(long m, double t) { this.modded = m; this.trueTotal = t; }
    }

    private static Result calculate(String s, char replace, int x, int y, long MOD) {
        long c0 = 0, c1 = 0, modded = 0;
        double trueTotal = 0;
        for (char c : s.toCharArray()) {
            if (c == '!') c = replace;
            if (c == '0') {
                modded    = (modded + (c1 % MOD) * (y % MOD)) % MOD;
                trueTotal += (double) c1 * y;
                c0++;
            } else {
                modded    = (modded + (c0 % MOD) * (x % MOD)) % MOD;
                trueTotal += (double) c0 * x;
                c1++;
            }
        }
        return new Result(modded, trueTotal);
    }
}
```
